Compression without sacrifice

XS3 is not summarization — it is re-encoding. Here is Matt Pocock's grilling skill — the interview primitive that stress-tests a plan before you build it — rewritten in XS3. Same instructions, every one intact:

mattpocock/skills · grilling · SKILL.md · plain English
Interview me relentlessly about every aspect of
this until we reach a shared understanding. Walk
down each branch of the decision tree, resolving
dependencies between decisions one-by-one. For
each question, provide your recommended answer.

Ask the questions one at a time, waiting for
feedback on each question before continuing.
Asking multiple questions at once is bewildering.

If a *fact* can be found by exploring the
environment (filesystem, tools, etc.), look it up
rather than asking me. The *decisions*, though,
are mine — put each one to me and wait for my
answer.

Do not act on it until I confirm we have reached
a shared understanding.

A communication language for multi-agent systems

AutoGen CrewAI LangGraph solved the routing of agent-to-agent messages.

The message itself is still ad-hoc JSON and free prose — no seat for uncertainty, no seat for law, no proof the meaning arrived intact.

XS3 is that missing wire format. Watch two agents talk ↓

🧠
PLANNER ASK
#m1 a ask; body {?f a fail; in "router.rs"; gate ?g}.
⌨️
TELL CODER
↩ re #m1 — your question, variables now bound
#m2 a tell; re #m1; body {#f1 a fail; in "router.rs"; gate g4; line 88 %95}.
🧠
PLANNER CONFIRM
↩ re #m2 — received, meaning verified
#m3 a confirm; re #m2; hash "9f3a".
// what just happened

The Planner asks which gate is making router.rs fail. The Coder answers — it's gate g4, at line 88 — and marks that answer 95% certain. The Planner confirms it received the exact same meaning by matching a hash of the message.

?f, ?g — the question is a pattern; the answer binds its variables %95 — uncertainty travels inside the message "9f3a" — the receipt is a hash of the meaning; agreement checked by machine

Any framework that passes strings between agents can carry XS3 today — no adapter, no SDK.
